利用github搭建自己的博客

2021年09月23日

今天开始就要在自己的hexo上发布博客了！

1. 新建博客

前提：

  1. 安装git
  2. 安装nodejs
  3. 安装hexo

开始之前先介绍一下heox的基本命令：

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

对应的缩写，比如：

hexo n == hexo new

hexo g == hexo generate

新建一个保存博客的存放目录，比如 E：\blog\Lovsog

进入到本地博客存放目录

初始化 个人博客

命令：hexo init

可以看到博客存放目录出现了如下内容：

2. 生成静态网页

hexo g #生成静态网页
hexo s #浏览

g后

s后

可以看到访问没问题。

但这个只是在本地预览，接着将这个博客部署到github。

3. 部署到github

新建一个github仓库

给仓库起个名称，注意！注意！注意！

仓库名称格式：用户名.github.io

因为这个仓库名称将作为你github博客的访问地址

编辑_config.yml，_config.yml在博客存放目录下

deploy:
  type: git
  repository: git@github.com:Art0white/Lovsog.github.io.git
  branch: main

repository仓库地址改为自己的

branch看自己的github仓库是master还是main

可以看到成功访问

注意！！！！！在设置域名是，用户名.github.io，这里的用户名必须是github的用户名，不能乱编

4. 编写博客

搭建好博客之后，日常中肯定是需要编写博客并且同步到github，接下来写一篇博客

新建一篇名为：helloworld的博客

命令：hexo new ‘helloworld’

可以看到生成了helloworld.md文件，通过这个md文件去写博客

默认的内容是这样的：

---
title: helloworld
date: 2021-09-22 20:36:45
tags:
---

编写好之后，开始部署到github

1.生成html文件

hexo g

2.上传到github

hexo d

再次访问就可以看到更新了
